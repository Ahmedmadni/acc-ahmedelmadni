# Accounting Editorial Design Skill

## Purpose

Design and implement a premium Arabic-first editorial website for Ahmed Elmadni.
The design combines financial professionalism with modern editorial web design.

## Visual Philosophy

Use contrast intentionally.

Light sections communicate:

- Clarity
- Openness
- Information
- Trust

Dark sections communicate:

- Authority
- Perspective
- Strong statements
- Premium positioning

Brown accents communicate:

- Warmth
- Human expertise
- Financial maturity

## Layout Philosophy

Do not make every section look like a card grid.

Use:

- Large typography
- Wide whitespace
- Asymmetric layouts
- Editorial compositions
- Overlapping elements
- Full-width statements
- Sticky content when useful
- Horizontal scrolling when appropriate
- Large visual anchors

Do not force every section into the same layout.

## Section Structure

A strong section may use:

1. Small eyebrow
2. Large headline
3. Supporting copy
4. Visual or data element
5. CTA

Do not force every section to use the same structure.

## Hero

The hero should immediately communicate:

- Who Ahmed is
- What he specializes in
- Why the visitor should care

Preferred narrative:

> "الأرقام لا تحتاج إلى مزيد من التعقيد.
> تحتاج إلى رؤية أوضح."

The hero should not be a generic centered SaaS hero.
Use a strong visual composition.

## Services

Organize services around client needs.

Recommended groups:

1. التقارير والقوائم المالية
2. محاسبة التكاليف والتحليل
3. الضرائب والزكاة
4. المحاسبة التشغيلية
5. الأنظمة والأتمتة

Avoid showing dozens of identical cards.

## Tools

Financial tools should feel useful and interactive.

Each tool should communicate:

- What it does
- Who it helps
- What result it produces

Use visual previews where possible.

## Portfolio

Projects should be presented as professional case studies.

Each project should communicate:

- Context
- Challenge
- Work
- Outcome

Avoid generic portfolio cards.

## Accounting Software

Use authentic brand assets already available or legally appropriate sources.
Do not use random placeholder icons.

Normalize logos using:

- Consistent containers
- Consistent visual scale
- Adequate contrast
- Accessible labels

## Interaction

Hover states should be subtle.

Preferred:

- Image movement
- Text shift
- Arrow movement
- Border transition
- Soft elevation

Avoid:

- Excessive scaling
- Aggressive rotation
- Shaking
- Flashing

## Design Test

Ask:

> "Would this section look intentional if all animation were removed?"

If not, the layout is not strong enough.
